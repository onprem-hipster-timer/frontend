# Code of Conduct

**[한국어 버전은 아래에 있습니다](#행동-강령)**

## Our Standards

This project maintains a professional, respectful community focused on technical excellence.

### Expected Behavior

- **Be respectful**: Treat everyone with respect. Disagreements about code are fine; personal attacks are not.
- **Be constructive**: Provide helpful feedback. "This doesn't work" is not useful; "This fails when X because Y" is.
- **Be professional**: Focus on the technical merits. Leave politics, religion, and unrelated topics elsewhere.
- **Be patient**: Not everyone has the same experience level. Help others learn.
- **Be concise**: Respect others' time. Get to the point.

### Unacceptable Behavior

- Personal attacks, insults, or derogatory comments
- Harassment of any kind
- Publishing others' private information
- Trolling or inflammatory comments
- Sustained disruption of discussions
- Any conduct inappropriate in a professional setting

### Technical Discussions

We value direct, honest technical feedback in the tradition of open source development:

- **Critique code, not people**: "This function has a bug" not "You wrote buggy code"
- **Be specific**: Point to exact lines, explain the issue, suggest alternatives
- **Accept criticism gracefully**: Your code is not you. Feedback helps us all improve.
- **Disagree professionally**: If you disagree with a review, explain why with technical reasoning

## Enforcement

### Reporting

Report unacceptable behavior to the project maintainers via:
- Email: jjh4450git@gmail.com
- Private message on GitHub

All reports will be reviewed and investigated. Reporters' identities will be kept confidential.

### Consequences

Project maintainers will take appropriate action for any behavior they deem inappropriate:

1. **Warning**: Private or public warning about the behavior
2. **Temporary Ban**: Temporary removal from project spaces
3. **Permanent Ban**: Permanent removal from project spaces

### Scope

This Code of Conduct applies to:
- Project repositories (issues, PRs, discussions)
- Project communication channels
- When representing the project publicly

# 행동 강령

## 우리의 기준

이 프로젝트는 기술적 우수성에 집중하는 전문적이고 존중하는 커뮤니티를 유지합니다.

### 기대하는 행동

- **존중하세요**: 모든 사람을 존중으로 대하세요. 코드에 대한 의견 충돌은 괜찮지만, 인신공격은 안 됩니다.
- **건설적으로**: 도움이 되는 피드백을 제공하세요. "이거 안 돼요"는 유용하지 않습니다. "X일 때 Y 때문에 실패합니다"가 좋습니다.
- **전문적으로**: 기술적 장점에 집중하세요. 정치, 종교, 관련 없는 주제는 다른 곳에서.
- **인내심을 가지세요**: 모든 사람이 같은 경험 수준을 가진 것은 아닙니다. 다른 사람이 배울 수 있도록 도와주세요.
- **간결하게**: 다른 사람의 시간을 존중하세요. 요점만 말하세요.

### 용납되지 않는 행동

- 인신공격, 모욕, 비하 발언
- 모든 종류의 괴롭힘
- 타인의 개인정보 공개
- 트롤링 또는 자극적인 댓글
- 토론의 지속적인 방해
- 전문적인 환경에서 부적절한 모든 행동

### 기술 토론

오픈소스 개발의 전통에 따라 직접적이고 솔직한 기술적 피드백을 존중합니다:

- **코드를 비판하세요, 사람을 비판하지 마세요**: "이 함수에 버그가 있습니다" O, "당신이 버그 있는 코드를 작성했습니다" X
- **구체적으로**: 정확한 줄을 가리키고, 문제를 설명하고, 대안을 제시하세요
- **비판을 우아하게 수용하세요**: 당신의 코드는 당신이 아닙니다. 피드백은 우리 모두가 발전하는 데 도움이 됩니다.
- **전문적으로 반대하세요**: 리뷰에 동의하지 않으면 기술적 근거로 이유를 설명하세요
